# Git Hygiene Rules

## AI Attribution Ban

Commits MUST NOT contain AI attribution. Before every commit, strip these patterns:

- `Co-Authored-By:` lines referencing Claude, Anthropic, or any AI assistant
- "Created by Claude" / "Generated by AI" / "AI-assisted" in commit messages
- Any trailer or line matching `Co-Authored-By:.*[Cc]laude`, `Co-Authored-By:.*[Aa]nthrop`

### Forbidden Patterns (case-insensitive)

```
Co-Authored-By: Claude
Co-Authored-By:.*Anthropic
Created by Claude
Generated by AI
AI-assisted
```

### Prevention

When creating commits:
1. Never add `Co-Authored-By` trailers referencing AI tools
2. Never include "Claude", "Anthropic", "AI-generated", or similar in commit messages
3. Write commit messages as if the developer wrote the code personally

### Detection and Cleanup

After committing, verify no attribution leaked:

```bash
git log main..HEAD --format="%B" | grep -iE "(claude|anthropic|co-authored-by.*ai|co-authored-by.*claude|co-authored-by.*anthrop|created by.*ai|generated by.*ai)"
```

If found, clean with:

```bash
git rebase main --exec 'MSG=$(git log -1 --format="%B" | grep -v "^Co-Authored-By:.*") && git commit --amend --reset-author --no-edit -m "$MSG"'
```

### Stacked Branch Cleanup

When cleaning attribution from stacked PRs, always start at the bottom of the stack and cascade-rebase upward. Child branches inherit parent commits — cleaning the base then rebasing propagates clean history without per-branch rewrites.

## Commit Message Style

- Use conventional commit format: `type: description` (e.g., `feat:`, `fix:`, `docs:`, `style:`, `refactor:`, `test:`, `chore:`)
- Keep subject line under 72 characters
- Write in imperative mood ("add feature" not "added feature")
- No AI-sounding language ("implement robust solution", "enhance functionality")
